## 📌 범례 및 분석 방법 설명

### 지표 방향 표시
| 기호 | 의미 |
|------|------|
| **[+]** | 값이 클수록 좋은 지표 (예: 인당 Commit수, 처리율 등) |
| **[-]** | 값이 작을수록 좋은 지표 (예: 이슈 처리일, 잔여이슈수 등) |

### 개선/악화 판정 기호
| 기호 | 의미 |
|------|------|
| ✅ 개선(유의) | 방향 개선 + 통계적 유의 (p < 0.05) |
| 🔵 개선(비유의) | 방향 개선 + 통계적 비유의 (p ≥ 0.05) |
| ⚠️ 악화(비유의) | 방향 악화 + 통계적 비유의 (p ≥ 0.05) |
| 🔴 악화(유의) | 방향 악화 + 통계적 유의 (p < 0.05) |

### p-value 설명
> **p-value**는 귀무가설(추세가 없다)이 참일 때 현재 데이터가 나올 확률입니다.
> - **유의 기준**: p < 0.05 (5% 수준) → 추세가 통계적으로 의미 있음
> - **비유의**: p ≥ 0.05 → 추세가 우연에 의한 것일 가능성 높음

### slope(기울기) 설명
> **slope**는 시계열 선형 회귀에서 **단위 주(week)당 지표값의 평균 변화량**을 의미합니다.
> - slope > 0: 시간이 지남에 따라 지표값이 **증가** 추세
> - slope < 0: 시간이 지남에 따라 지표값이 **감소** 추세
> - [+] 지표에서 slope > 0이면 개선 추세, slope < 0이면 악화 추세
> - [-] 지표에서 slope < 0이면 개선 추세, slope > 0이면 악화 추세
> - |slope| 값이 클수록 변화 속도가 빠름

### UNIT 건강도 점수 계산 방식
> **점수화 기준**: 전주 대비 변화 방향 + 선형 회귀 p-value (유의성)
> 
> | 상태 | 점수 |
> |------|------|
> | 악화(유의): 방향 악화 + p < 0.05 | **-5점** |
> | 악화(비유의): 방향 악화 + p ≥ 0.05 | **-3점** |
> | 개선(비유의): 방향 개선 + p ≥ 0.05 | **+1점** |
> | 개선(유의): 방향 개선 + p < 0.05 | **+2점** |
> 
> **종합 건강도 점수 = Σ(각 지표별 점수) / 유효지표수 × 10** (표준화)
> - 각 지표에 대해 시계열 기울기(slope)의 방향과 p-value를 계산하여 위 기준으로 점수 부여
> - 점수가 높을수록 건강한 상태, 낮을수록 개선 필요
