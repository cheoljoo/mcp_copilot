## 📊 분석 안내

### 지표 방향 표시
- **[+]** : 값이 **높을수록** 좋은 방향 (예: 처리율, 활성개발자수)
- **[-]** : 값이 **낮을수록** 좋은 방향 (예: 이슈처리일, 잔여이슈수)

### OLS 회귀(Ordinary Least Squares Regression) 란?

> **OLS(최소제곱법) 선형회귀**는 시간(날짜)과 지표값 사이의 직선 관계를 수학적으로 찾는 방법입니다.
>
> **핵심 개념**: 각 날짜의 실제 관측값과 직선 예측값의 차이(오차)를 최소화하는 직선 `y = a·x + b` 를 구합니다.
> - `x` : 날짜 순서(0, 1, 2, … 일 경과)
> - `y` : 지표값
> - `a` = **slope(기울기)** : 하루당 평균 변화량
> - `b` = intercept(절편) : 시작 시점 예측값
>
> **왜 OLS를 쓰나?** 단순히 최근값과 이전값만 비교하면 일시적 변동(노이즈)에 민감합니다.
> OLS는 전체 기간의 데이터를 한꺼번에 고려해 **전반적인 장기 추세**를 파악하므로 더 안정적입니다.
>
> **예시**: `slope = +0.05` 이고 지표가 `[-](낮을수록 좋음)` 이면 → 하루에 0.05씩 증가 → **악화 추세**
> **예시**: `slope = -0.02` 이고 지표가 `[-](낮을수록 좋음)` 이면 → 하루에 0.02씩 감소 → **개선 추세**

### 통계 용어 설명
| 용어 | 설명 |
|------|------|
| **OLS 회귀** | 전체 기간 데이터를 활용해 시간에 따른 장기 추세를 직선으로 추정하는 방법 (Ordinary Least Squares) |
| **slope (기울기)** | OLS 직선의 기울기. 하루당 지표 변화량. [+] 지표에서 slope > 0 → 개선, [-] 지표에서 slope < 0 → 개선 |
| **p-value (유의확률)** | slope가 통계적으로 의미 있는지 나타내는 값. **p < 0.05이면 유의** (95% 신뢰수준), p ≥ 0.05이면 비유의. 즉 p < 0.05 일 때만 해당 추세를 신뢰할 수 있음 |
| **유의(p<0.05)** | 관찰된 추세가 우연이 아닐 확률이 95% 이상. 해당 slope를 신뢰할 수 있음 |
| **비유의(p≥0.05)** | 추세가 통계적으로 불분명함. 방향은 참고만 가능하며 과신하면 안 됨 |

### UNIT 점수 계산 기준
각 지표별로 slope 방향과 p-value를 결합하여 점수를 부여, 합산합니다.

| 상태 | 조건 | 점수 |
|------|------|------|
| 개선(유의) | (slope × 지표부호) > 0 AND p < 0.05 | **+2** |
| 개선(비유의) | (slope × 지표부호) > 0 AND p ≥ 0.05 | **+1** |
| 악화(비유의) | (slope × 지표부호) < 0 AND p ≥ 0.05 | **-3** |
| 악화(유의) | (slope × 지표부호) < 0 AND p < 0.05 | **-5** |

> **계산식**: 각 UNIT의 건강도 점수 = Σ(전체 지표별 score)  
> 점수가 높을수록 전반적으로 개선 추세, 낮을수록 악화 추세를 의미합니다.
